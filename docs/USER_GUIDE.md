# 数据源可靠性调研与适配计划

## 1. 当前免费接口稳定性判断
当前项目主要使用 AkShare 作为免费数据源入口。AkShare 本身非常适合快速开发和个人研究，但在长期产品化场景下存在这些典型风险：

- 上游网页或接口字段变化会影响可用性
- 无明确 SLA（服务等级承诺）
- 网络波动、限流、反爬、页面结构变化都可能导致失败
- 某些接口更适合研究，不一定适合稳定批量生产使用

结合当前项目已有真实运行记录，已经出现过：
- `RemoteDisconnected`
- 连接中断
- 实时拉数失败导致回测中止

结论：
**AkShare 适合作为默认免费入口，但不适合作为唯一长期稳定数据源。**

## 2. 数据源选择原则
后续适配数据源时，建议从这些维度评估：

1. **可靠性**：是否有稳定接口、是否经常变动、是否有 SLA
2. **覆盖范围**：A 股日线、实时行情、指数、板块、财务、成分股等是否齐全
3. **成本**：免费、积分制、订阅制、企业报价
4. **接入难度**：Python SDK / HTTP API 是否清晰
5. **使用限制**：频率限制、商用限制、需要实名或审核与否
6. **数据一致性**：字段是否稳定，是否适合长期维护

## 3. 可考虑的数据源候选

### 3.1 AkShare
- 类型：免费开源聚合工具
- 优点：免费、上手快、覆盖广、适合原型和研究
- 缺点：上游依赖强、无 SLA、稳定性受外部网页变化影响较大
- 适合阶段：MVP / 个人研究 / 免费默认入口

### 3.2 Tushare Pro
- 类型：国内常用量化数据服务
- 优点：A 股生态常见，文档和字段体系较成熟，适合量化研究
- 缺点：很多数据依赖积分或付费等级，实时能力和频率受限制
- 价格特点：基础功能可免费/积分获取，更高额度通常需要付费或较高积分门槛
- 适合阶段：个人进阶用户、低到中频量化工具

### 3.3 JoinQuant / 聚宽（JQData）
- 类型：量化平台数据服务
- 优点：量化生态成熟，研究环境和数据体系较完整
- 缺点：部分能力与平台体系绑定较深，纯外部产品接入时需关注授权方式和费用
- 价格特点：一般按产品/权限等级区分，个人与机构差异较大
- 适合阶段：量化研究、策略验证、进阶用户

### 3.4 Ricequant / 米筐（RQData）
- 类型：量化数据与研究平台
- 优点：接口较体系化，量化研究常见
- 缺点：产品能力与授权范围需具体确认，成本通常高于免费源
- 价格特点：偏商业化，具体价格多需咨询
- 适合阶段：中高阶用户、专业量化场景

### 3.5 Wind / iFinD / 同花顺金融数据终端
- 类型：专业金融终端/机构数据服务
- 优点：稳定性、覆盖深度、机构认可度高
- 缺点：价格高、授权复杂、不适合普通个人用户直接承担
- 价格特点：通常明显高于个人可接受范围，多为机构采购
- 适合阶段：机构版、企业版、高可靠性要求场景

### 3.6 东方财富 / 新浪 / 腾讯等直接网页/接口抓取
- 类型：免费网页级来源
- 优点：免费，信息多
- 缺点：最不稳定，最容易因页面变化失效；商用和长期维护风险高
- 适合阶段：仅临时补充，不建议作为主力稳定源

## 4. 综合建议

### 短期（当前最现实）
- 继续保留 AkShare 作为默认免费入口
- 优先增强：缓存、重试、超时、失败降级、离线优先
- 增加 provider router，允许后续无痛切换数据源

### 中期（最推荐）
- 新增 Tushare Pro provider
- 配置中增加 API Key / token 支持
- 让用户可选：
  - 免费模式：AkShare
  - 稳定增强模式：Tushare Pro

### 中高阶阶段
- 评估 JQData / RQData 的授权与接入成本
- 面向进阶量化用户做更专业的数据适配包

### 长期
- 如果产品面向企业或机构，再评估 Wind / iFinD 级别数据源

## 5. 适配计划

### Phase A：提升现有免费源可靠性
1. 为 provider 增加统一重试机制
2. 增加超时控制
3. 增加失败时优先读取缓存的降级策略
4. 区分“实时数据失败”和“历史缓存可用”的场景
5. 在 CLI/UI 中给出更明确的降级提示

### Phase B：新增可插拔付费/稳定数据源能力
1. 新增 `DataSourceConfig` 中的 token/api_key 字段
2. 实现 `TushareProvider`
3. 在 `data/providers/router.py` 支持按配置切换
4. 补 provider 级测试和兼容层
5. 更新文档说明不同数据源的适用人群

### Phase C：多数据源优先级与回退机制
1. 支持主数据源 + 备用数据源
2. 某类数据（如实时行情）失败时自动尝试 fallback provider
3. 记录数据源命中和失败日志
4. 支持按功能维度路由，例如：
   - 历史日线用 Provider A
   - 实时行情用 Provider B

## 6. 推荐路线
最推荐的现实路线是：

- **当前默认**：AkShare
- **下一步增强**：AkShare + 更强缓存/重试/降级
- **第一稳定付费扩展**：Tushare Pro
- **后续专业扩展**：JQData / RQData
- **机构级扩展**：Wind / iFinD

这样可以兼顾：
- 普通用户成本
- 工程实现难度
- 长期扩展性
- 稳定性提升路径


## 7. 测试 token 验证结论
使用用户提供的测试 token 进行了最小化接口验证，结果表明：
- 已成功连通 Tushare Pro 服务本身
- 但测试 token 对 `index_daily` 和 `daily` 接口均提示“没有接口访问权限”

这说明：
1. Tushare Pro 的可用性不仅取决于 token 是否有效，还取决于具体接口权限
2. 后续适配时需要在文档中明确区分“已配置 token”和“具备目标接口权限”是两件事
3. 实现上应对权限不足给出清晰提示，而不是简单当作网络错误处理

- 如果返回“token 不对，请确认”，通常表示 token 无效、输入错误或已失效。
- 如果返回“没有接口访问权限”，通常表示 token 有效，但目标接口权限不足。

### Tinyshare 授权码怎么配？
如果你买的是 Tinyshare 的代理/兼容服务：
- `data_source.provider: tushare`
- `data_source.tushare_sdk: tinyshare`
- `data_source.tushare_token: 你的授权码`

这样程序内部会用 `tinyshare` SDK，但仍走 Tushare 风格接口。

## 2️⃣2️⃣ 数据源诊断命令
可运行：
```bash
ashare-strategy doctor-data --output json
```
它会检查：
- 当前数据源类型
- SDK 类型（如 tinyshare）
- token/授权码是否可用
- 核心接口是否可访问
- 是否只能依赖缓存降级

如果你在排查“为什么今天拉不到数据”，建议先跑这个命令。
