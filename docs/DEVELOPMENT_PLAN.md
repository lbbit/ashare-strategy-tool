# 开发计划

## 总体目标
本项目的长期目标不是只做一个简单回测脚本，而是逐步演进为一个：
- 面向普通股民也容易使用的 A 股辅助决策工具
- 同时支持 CLI、图形界面、AI Agent 集成
- 先支持手工持仓分析与换仓建议
- 后续可扩展到量化 API / 券商 API 接入
- 支持多数据源（免费 / 付费 / API Key）切换
- 支持用户自定义参数、策略模板、回测和执行流程

## 阶段规划

| 阶段 | 目标 | 关键任务 | 状态 |
|---|---|---|---|
| Phase 1 | 项目初始化 | 建立包结构、配置系统、CLI/UI 框架 | 已完成 |
| Phase 2 | MVP 数据与策略闭环 | 接入免费数据源、缓存、板块/个股筛选、基础买卖规则 | 已完成 |
| Phase 3 | 回测与分析 | 组合回测、滚动回测、指标统计、报告导出 | 已完成基础版 |
| Phase 4 | 用户执行闭环 | 持仓持久化、每日交易计划、CLI/UI 结果查看 | 已完成基础版 |
| Phase 5 | 工程化交付 | 测试、文档、Windows 打包、Release 自动化 | 已完成基础版 |
| Phase 6 | 架构升级 | 服务拆分、数据源抽象、配置分层、账户抽象 | 进行中 |
| Phase 7 | 多数据源扩展 | 支持可插拔数据源，预留 API Key 配置能力 | 待开发 |
| Phase 8 | 多账户/同步扩展 | 手工持仓、JSON 持仓、券商/量化 API 账户统一建模 | 待开发 |
| Phase 9 | 高级交互层 | Agent-friendly CLI、增强 UI、API 层预留 | 待开发 |
| Phase 10 | 自动化与实盘扩展 | 调度、通知、风险控制、实盘执行适配层 | 待开发 |

## 当前阶段重点：Phase 6 架构升级

### 目标
在不破坏现有功能可用性的前提下，为后续扩展打基础。

### 关键任务
1. 抽象统一数据源接口
2. 拆分当前 `TradingService` 为多个单一职责服务
3. 拆分配置模型：策略配置 / 数据源配置 / 应用配置
4. 设计统一账户与持仓模型
5. 规划 CLI 稳定 JSON 输出格式
6. 保持 Streamlit UI 可用，同时避免业务逻辑写死在 UI 中

## 面向用户的长期能力目标

### 普通用户模式
- 手工录入持仓
- 自动生成分析与换仓建议
- 本地保存持仓与计划
- 简单安装、简单使用、界面清晰

### 进阶用户模式
- 自定义参数
- 自定义策略模板
- 更稳定的数据源
- 多版本回测比较

### 量化接入模式
- 支持外部 API Key
- 支持账户读取
- 支持交易执行适配层
- 支持模拟盘 / 实盘切换

## 技术路线建议

### 数据源
- 短期：AkShare 为主
- 中期：增加统一 provider 接口
- 后期：支持付费 / API Key 数据源

### 账户体系
- 短期：本地 JSON + 手工输入
- 中期：统一账户抽象层
- 后期：接券商 / 量化 API

### UI
- 短期：继续使用 Streamlit
- 中期：增加更强的页面组织与交互
- 后期：根据需求评估独立前端或桌面端方案

### CLI
- 短期：保持易用
- 中期：增加机器可读输出
- 后期：支持 AI Agent 更稳定调用
